Trip workspace

This app now loads trips through a manifest, so new destinations can become separate saved trip files without rewriting the map.

Route loading

Preparing road-following routes…

Each coloured route is requested from OSRM in your browser, so it follows the road network rather than drawing straight lines. If the routing server is unavailable, the map keeps a dashed fallback line through the planned stops and still shows the road-number/type labels.

Vehicle and energy estimate

Costs use the loaded route distance when available. Before OSRM finishes, estimates use the planned stop-to-stop fallback distance.

Place layers

These are static planning markers for now. Later they can be replaced with live charger, hotel, and activity datasets.
